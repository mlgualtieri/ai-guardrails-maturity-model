# Changelog

All notable changes to the AI Guardrails Maturity Model are recorded here. The model follows a
dated, versioned release so that readers can track how the controls, levels, and sourcing evolve
over time.

## v1.0 — Initial release (July 2026)

The first public release of the AI Guardrails Maturity Model.

- **36 controls across 13 categories**, each assigned to the maturity level at which it should first
  appear.
- **Four maturity levels** — Basic, Intermediate, Advanced, and Leading Edge — with a coherent
  posture statement and a stated gap to the next level for each.
- **Two applicability tags** (`Both`, `Agentic`) and an alert-fatigue rating on every control, so a
  reader can tell quickly what applies to their deployment and what to expect operationally.
- **A primary authoritative source on every control**, drawn from NIST, OWASP, MITRE, CISA/NCSC, the
  Cloud Security Alliance, ISO/IEC, the FinOps Foundation, and peer-reviewed research. See the
  [Reference Library](reference/sources.md).
- **Full interlinking** — the catalog links to every control, each control links to its level and
  its related controls, and each level links to the controls that first appear there.

---

*Future releases will be recorded here as the field, the underlying standards, and the threat
landscape move. The model is a snapshot of a fast-moving area, and it is expected to change.*
